About

A builder with a taste for live systems.

This page avoids fake résumé gravity. It gives visitors a fast, memorable read: what Tianyi makes, how he thinks, and where to go next.

I like software that feels inhabited: fast enough to disappear, visual enough to remember, and honest about what it can do.
01
Build the useful thing first. Interfaces should start with the actual workflow, not a decorative cover page. The first viewport here already has data, navigation, and identity.
02
Let projects become instruments. Games, music tools, dashboards, and AI helpers all share the same requirement: they need feedback loops that feel clear and immediate.
03
Keep deployment boring. The site is one deployable file today, with clean upgrade paths to Vercel, Netlify, GitHub Pages, Astro, or Next.js when the content grows.
Lab

Small modules worth stealing from yourself.

Each module is designed as a reusable piece, so the personal site can become a playground instead of a static brochure.

Weather skin

Live temperature and sky state from Open-Meteo, using visitor geolocation when available and Paris as a graceful fallback.

Signal canvas

A lightweight canvas visual that reacts to time, theme, and viewport size without blocking the page.

GitHub trace

Public profile data is pulled at runtime, making the site feel connected to ongoing work without manual edits.

Theme memory

A restrained light and night palette with local preference storage, visible focus states, and reduced-motion support.
